    Référence API¶

    Cette page documente tous les endpoints de l'API REST AdvancedDevSample.

    Base URL¶

    http://localhost:5069/api

    Endpoints disponibles¶

    Produits¶

    • GET /products - Lister tous les produits
    • GET /products/{id} - Obtenir un produit par ID
    • POST /products - Créer un nouveau produit
    • PUT /products/{id}/price - Modifier le prix d'un produit
    • POST /products/{id}/activate - Activer un produit
    • POST /products/{id}/deactivate - Désactiver un produit

    Commandes¶

    • GET /orders/{id} - Obtenir une commande par ID
    • POST /orders - Créer une nouvelle commande
    • POST /orders/{id}/lines - Ajouter une ligne à une commande
    • PUT /orders/{id}/lines - Modifier la quantité d'une ligne
    • DELETE /orders/{id}/lines/{productId} - Supprimer une ligne
    • POST /orders/{id}/confirm - Confirmer une commande
    • POST /orders/{id}/cancel - Annuler une commande

    Clients¶

    • GET /customers - Lister tous les clients
    • GET /customers/{id} - Obtenir un client par ID
    • POST /customers - Créer un nouveau client
    • PUT /customers/{id} - Mettre à jour un client
    • DELETE /customers/{id} - Supprimer un client
    • POST /customers/{id}/activate - Activer un client
    • POST /customers/{id}/deactivate - Désactiver un client

    Fournisseurs¶

    • GET /suppliers - Lister tous les fournisseurs
    • GET /suppliers/{id} - Obtenir un fournisseur par ID
    • POST /suppliers - Créer un nouveau fournisseur
    • PUT /suppliers/{id} - Mettre à jour un fournisseur
    • DELETE /suppliers/{id} - Supprimer un fournisseur
    • POST /suppliers/{id}/activate - Activer un fournisseur
    • POST /suppliers/{id}/deactivate - Désactiver un fournisseur

    Codes de statut HTTP¶

    Code Signification Utilisation
    200 OK Requête réussie (GET, PUT)
    201 Created Ressource créée (POST)
    204 No Content Opération réussie sans contenu de retour
    400 Bad Request Données invalides ou violation de règle métier
    404 Not Found Ressource non trouvée
    500 Internal Server Error Erreur serveur

    Format des erreurs¶

    Toutes les erreurs retournent un objet JSON avec les champs suivants :

    {
  "title": "Type d'erreur",
  "detail": "Message d'erreur détaillé"
}

    Exemples¶

    Erreur de validation (400) : 

    {
  "title": "Erreur métier",
  "detail": "Le prix doit être strictement positif."
}

    Ressource non trouvée (404) : 

    {
  "title": "Ressource introuvable",
  "detail": "Produit non trouvé."
}

    Authentification¶

    Note

    Cette version de l'API ne nécessite pas d'authentification. C'est une application de démonstration.

    Swagger UI¶

    L'API est documentée avec Swagger/OpenAPI. Accédez à l'interface interactive à :

    http://localhost:5069/swagger

    Swagger UI vous permet de : - Visualiser tous les endpoints - Tester les API directement - Voir les schémas de requêtes et réponses - Télécharger la spécification OpenAPI
